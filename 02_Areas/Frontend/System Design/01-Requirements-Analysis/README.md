# Requirement Analysis in Frontend System Design

## Overview
Before drawing a single component tree or debating Redux vs. Context, you must understand **what** you are building and **who** you are building it for. 

Requirement Analysis is the crucial first step in any system design process. It is the act of translating a vague prompt (e.g., "Design Facebook") into a concrete, actionable engineering specification.

If you fail this step, you will architect the wrong system.
